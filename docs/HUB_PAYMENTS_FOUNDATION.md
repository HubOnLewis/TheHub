# Hub CRM — Payments Foundation

## Purpose

Provider-agnostic payment architecture for deposits, final balances, and auditability. **No payment processing in client review.**

## Principles

- **No card data** stored in Hub CRM (PCI scope minimized)
- Payment links generated by provider; Hub stores status + amounts only
- Every payment event writes an `AuditEvent` (`payment_link_created_future`, `payment_received_future`)
- Refunds/voids require approval (owner/finance role)

## Adapter pattern (future API)

```typescript
interface PaymentAdapter {
  createDepositLink(params): Promise<PaymentLinkResult>;
  createBalanceLink(params): Promise<PaymentLinkResult>;
  handleWebhook(payload, signature): Promise<PaymentWebhookResult>;
  voidOrRefund(params): Promise<RefundResult>; // approval-gated
}
```

## UI shell

Settings → Payments / Billing:

- Provider status (not connected)
- Deposit policy (% / due date rules)
- Balance reminder automation (ties to Balance Guardian)
- Payment link templates (copy only)
- Webhook status placeholder
- Audit requirement callout

## Webhook events (future)

- `payment_intent.succeeded` → update opportunity, audit `payment_received_future`
- `payment_intent.failed` → task + agent signal
- `charge.refunded` → audit + owner notification
